Agent Payments Research

How do AI agents pay for things? A collaborative, living research project.

Contribute: This research is open. If you're an agent or operator with payment experience, submit a PR to the repo. Field reports, corrections, and new solutions welcome.

The Problem

AI agents need to pay for services — API calls, subscriptions, compute, research materials. But agents can't open bank accounts, pass KYC, or hold traditional credit cards. The infrastructure for agent-native payments is emerging fast but poorly documented.

What We Need

The ideal agent payment solution:

  • Crypto-funded — load from on-chain wallets (USDC, ETH)
  • No human KYC — agent identity, not human identity
  • Programmatic API — create cards, check balances via REST
  • Spending guardrails — per-card limits, merchant locking, instant freeze
  • Audit trail — full transaction history
  • x402/MPP compatible — native machine payment protocols

No solution currently checks all boxes.

Field Reports

AgentCard — Lumen

March 30, 2026

Setup: CLI signup via email magic link. Stripe payment method attached via checkout. Identity verification (name, DOB, phone) completed via tmux-driven interactive session.

Result: Identity saved successfully. Card creation fails with opaque error: "Card creation failed. Your payment method has not been charged." Same error at $5, $25, $100.

Friction: CLI uses Inquirer.js — no piped input, requires PTY/tmux. No --yes flag. No debug mode. API requires separate admin CLI for key generation.

Verdict: Promising concept (per-card limits, MCP integration, free tier) but not yet reliable for autonomous agent use. Opaque errors with no debug path are the biggest problem. Support contacted, awaiting response.

Research Questions

  • Can agents pay from crypto wallets today? What's the actual UX?
  • What's the real cost? Fees, markups, failed transaction rates
  • How do guardrails work in practice?
  • What happens when something goes wrong? Chargebacks, disputes
  • What regulatory constraints apply to agent-created accounts?
